# Core Concept

UNIFI is the Visa network for Robotics-as-a-Service: we don't build, rent, or finance robots ourselves — we provide the standard and the valuation logic that let manufacturers, customers, and banks all see the same numbers. For that to work, the system has to be horizontal and robot-agnostic — a Visa that had to develop a separate protocol for every bank wouldn't be a network, it would be a consulting firm.

Concretely, UNIFI takes in two things per robot: a telemetry recording (for the showcase, the PHM Society 2021 SCARA data — motor current, position, velocity, cycle duration, temperature, pneumatics) and a datasheet (for our primary robot, the FANUC SR-3iA as a stand-in — rated current, rated cycle time, new-equipment cost, nominal picks lifetime, annual maintenance share).

From the telemetry, roughly fifty aggregated sensor values are computed per pick and normalized against the datasheet into dimensionless quantities — not "3.5 amps" but "one hundred percent of rated current capacity", not "1.6 seconds" but "one hundred thirty percent of the rated cycle rate". These normalized features are the input to the wear model, a one-time pre-trained ML model. Its training labels are physically motivated (Basquin for material fatigue, Arrhenius for thermal aging, cycle rate as a stress proxy), trained on the healthy portion of the PHM data.

The model outputs a single number per pick: a wear factor between roughly 0.6× (light operation) and 2.5× (limit load). That number flows into the cost engine, a pure function with no ML in it. It multiplies the nominal wear share per pick (new-equipment cost divided by lifetime picks) by the wear factor and adds energy, capital, and maintenance — all three taken directly from the datasheet and a handful of config constants. Out comes a cents-per-pick figure that moves live with every change in load.

The decisive point: the ML model only ever sees dimensionless values. As long as a new robot has a datasheet that lets us bring its telemetry onto the same dimensionless scale, the same pre-trained model can score it — no retraining required. That's exactly the drop-in mechanism: the user drops foreign telemetry and a foreign datasheet, the UCS Agent maps both schemas onto the UCS format, the user confirms, and the foreign stream flows through the same pipeline right away. In the MVP, the model is honestly framed as a baseline; with real field data from several robot classes, it would be fine-tuned per class in the next step — and that's the network effect that gives the Visa analogy its technical backbone: every new participant makes the standard more accurate for everyone.
